### **القائمة المرتبطة المفردة (Singly Linked List)**

```
[HEAD] → [Student 1] → [Student 2] → [Student 3] → [Student N] → null
  │          │              │              │               │
  │          ▼              ▼              ▼               ▼
  │    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
  │    │ ID: 1001│    │ ID: 1002│    │ ID: 1003│    │ ID: 100N│
  │    │ Name    │    │ Name    │    │ Name    │    │ Name    │
  │    │ Major   │    │ Major   │    │ Major   │    │ Major   │
  │    │ next →  │    │ next →  │    │ next →  │    │ next →  │
  │    └─────────┘    └─────────┘    └─────────┘    └─────────┘
  │
  ▼
null (إذا كانت القائمة فارغة)
```

## 🔄 مخطط تدفق العمليات (Operations Flow)

### **1. عملية الإضافة (Insert)**
```
المستخدم → UniversityApp → StudentList.insert() → إنشاء Student جديد → إضافة للقائمة
    │              │                   │                        │
    ▼              ▼                   ▼                        ▼
إدخال بيانات   التحقق من صحة   التحقق من عدم تكرار   إنشاء عقدة جديدة
   ID, Name       الإدخال          الـ ID              وربطها بالقائمة
```

### **2. عملية البحث (Search)**
```
المستخدم → UniversityApp → StudentList.search() → التجوال في القائمة → العثور على Student
    │              │                   │                        │
    ▼              ▼                   ▼                        ▼
إدخال ID     التحقق من الرقم   مقارنة ID مع كل عقدة   إرجاع Student أو null
```

### **3. عملية الحذف (Delete)**
```
المستخدم → UniversityApp → StudentList.delete() → البحث عن Student → تعديل الروابط
    │              │                   │                        │
    ▼              ▼                   ▼                        ▼
إدخال ID     التحقق من الرقم   تحديد موقع العقدة   ربط السابق بالتالي
```

## 📈 مخطط تعقيد العمليات (Complexity Analysis)

| العملية | تعقيد الوقت | تعقيد المساحة | الشرح |
|---------|-------------|---------------|-------|
| **Insert** | O(n) | O(1) | التجوال حتى نهاية القائمة |
| **Display** | O(n) | O(1) | التجوال في جميع العقد |
| **Search** | O(n) | O(1) | التجوال حتى العثور على العنصر |
| **Update** | O(n) | O(1) | بحث + تعديل |
| **Delete** | O(n) | O(1) | بحث + تعديل الروابط |

## 🎯 نقاط القوة والضعف

### **نقاط القوة:**
- ✅ استخدام هيكل بيانات ديناميكي (لا حاجة لحجم مسبق)
- ✅ كفاءة في الإضافة والحذف (مقارنة بالمصفوفات)
- ✅ تنظيم الكود في فئات منفصلة
- ✅ واجهة رسومية سهلة الاستخدام

### **نقاط الضعف:**
- ❌ البحث بطيء (O(n))
- ❌ لا يمكن الوصول العشوائي للعناصر
- ❌ تحتاج مساحة إضافية للمؤشرات
